1. `recv/AccountDAO.createAccountInTx()`
   - 부서 FK 검증 -> 없으면 `throw new SQLException("DEPT_NOT_FOUND:" + deptCd)`

   - account + account_history 두 INSERT
2. `recv/InboxDAO.insertInTx()` — UNIQUE 제약 의존
3. `recv/AccountBatchServlet.java` — **이 모듈 핵심**

   - accountDAO 호출 (트랜잭션 외부에서 conn 넘김)
   - 예외별 errCode 분기: DEPT_NOT_FOUND->E30, REQUIRED_MISSING->E10, EMP_ID_FORMAT->E32

## Step 5 — 모니터링 (2개)
1. `monitor/IfMonitorServlet.java` — outbox + inbox + account
2. `monitor/IfRetryServlet.java` — F->N

## 자주 하는 실수
- 비밀번호 평문 저장 — pwd_hash 컬럼은 SHA-256 hex 64자
- pwd_hash 를 모니터링 화면에 노출 — 절대 금지 (CHECKLIST 항목 13)
- deptCd 미존재인데 account 만 INSERT 되고 마스터는 깨짐 -> 부서 FK 검증을 트랜잭션 안에서
- inbox INSERT 를 account 차감 뒤에 -> 정상 처리됐는데 inbox 가 안 들어가는 케이스
- 멱등 스킵 시 outbox 를 S 로 안 바꾸면 -> 무한 루프
